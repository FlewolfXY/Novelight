# Novelight 📖✨

**一本活着的人生之书 —— 永不遗忘的 AI 自我理解系统。**

Novelight 是一个 AI 系统，它持续地将你的对话、反思、健康数据和生命事件编织成一份不断演化的、持久的自我理解——服务于个人成长，也可以选择性地让心理咨询的效率提升十倍。

---

## 为什么 Novelight 需要存在

每年，无数人走进咨询室，试图把几个月甚至几年的生活经历压缩进50分钟的会谈里。咨询师拿到的是一个被扭曲的、不完整的碎片。来访者觉得自己怎么都说不清楚真正经历过的东西。

与此同时，人们把最深层的反思倾倒进 AI 对话里，然后上下文被清除，一切从零开始。他们用可穿戴设备追踪心率和睡眠，但完全不知道这些数据在心理层面*意味着*什么。他们写了一周的日记然后放弃了，因为维护成本太高。

**自我理解的基础设施是断裂的。**

- AI 对话强大但短暂 —— 上下文清除后，一切从零开始。
- 日记 App 是被动的 —— 它们等你来写，但不理解你写了什么。
- 心理健康 App 是通用的 —— 对复杂创伤患者和工作压力大的白领说一样的话。
- 可穿戴设备数据没有意义 —— 知道心率降到了基线以下，但没有人告诉你"你的身体在说这个人是安全的"。
- 咨询师很专业但信息匮乏 —— 他们把宝贵的会谈时间花在收集历史信息上，而不是做真正的治疗。

Novelight 填补了所有这些之间的裂缝。它是那个**记住一切、连接一切、翻译一切**的持久层。

## Novelight 是什么（不是什么）

| Novelight 是 | Novelight 不是 |
|---|---|
| 关于你的持久理解，不会因为模型更换而消失 | 一个聊完就忘的聊天机器人 |
| 一本有章节和转折点的、关于你人生的活叙事 | 一个等你来写的日记 App |
| 连接你的真实经历和你的咨询师的桥梁 | 心理咨询的替代品 |
| 用心理学语言解读你身体数据的翻译器 | 一个健身追踪器 |
| 跨越数月和数年的人生模式识别器 | 单次对话的分析工具 |

**一句话：Novelight 让每个人在15岁就能开始做大多数人到中年危机才被迫开始做的事情。**

## 核心架构：三层模型

```
第一层：自然流（持续感知）
    ↓  提取 / 标注
第二层：河床（持久理解）
    ↓  综合 / 叙事
第三层：书（叙事生成）
```

### 第一层 —— 自然流（The Stream）

一个人生活的原始数据流。Novelight 通过以下方式接收数据：

- **对话：** 用户随时可以和 Novelight 聊天。不需要结构化输入。"他今天给我买了薄荷糖，我哭了"就够了。
- **授权数据接入：** 聊天记录导入、可穿戴设备数据同步（心率、睡眠、压力、运动）、照片上传。
- **主动感知：** 当 Novelight 检测到异常模式时（比如连续三天睡眠数据异常差），它会温和地询问："最近有什么在困扰你吗？"

**设计原则：采集必须极轻。** 如果它让人觉得像工作，人们就会停止使用。用户唯一的任务就是活着。

### 第二层 —— 河床（The Riverbed）

这是 Novelight 真正的核心。一个持久的、结构化的、持续更新的关于一个人的理解模型。

**关键特性：理解独立于任何一个大语言模型（LLM）存在。**

当前的 AI 对话把理解储存在上下文窗口里。上下文清除 → 理解消失。Novelight 将理解提取到一个**独立的持久层**（向量数据库 + 结构化知识图谱 + 摘要系统），任何模型都可以查询这个持久层。**模型可以换，理解不会丢。**

河床包含什么：

- **事件图谱：** 生命中的关键事件及其关联。不是时间线，是网络。"初中朋友的背叛"和"伴侣不回消息时的恐慌"之间有一条连线。这些连接随着新数据持续被发现和更新。

- **模式库：** 反复出现的行为、情绪和关系动力学。每个模式包含：首次观察时间、重复出现记录、进化轨迹。

  ```
  模式："被悬置时的灾难化思维"
  首次观察：2026-01-16
  重复出现：2026-03-07, 2026-03-21, 2026-03-22
  触发条件：伴侣在共同决策场景中延迟回复超过1小时
  进化轨迹：强度递减。恢复时间从数天缩短到数小时。
            再过一个月就消失了。
  ```

- **人物画像：** 相对稳定的特征 —— 人格结构、依恋类型、核心创伤、防御方式、力量来源。不是固定的，随新数据缓慢更新。

- **进化轨迹：** 最有价值的部分。不是"你是什么样的人"，而是"你正在变成什么样的人"。

  ```
  2025年11月："他不回消息就是要抛弃我。"
  2026年1月："他不回消息。我需要分析原因。问问AI。"
  2026年3月："他不回消息。大概安息了吧。反正跑不掉。"
  ```

  这条轨迹，附带时间戳和支撑数据，对咨询师来说是无价之宝。

### 第三层 —— 书（The Book）

用户和咨询师直接看到的东西。

Novelight 将第二层的理解渲染成**一本有章节的书** —— 不是数据仪表盘，是叙事。

每一章包含：

- **叙事摘要：** 不是日志条目，是故事。*"这是她终于不再为自己的需求感到羞耻的那个月。"*
- **锚点：** 那个时段里最重要的时刻 —— 转折、触发、洞察、身体信号。
- **支撑素材：** 叙事生长出来的原始对话片段、照片、身体数据。用户控制哪些对谁可见。
- **空白：** 没有数据的时段。沉默本身就是叙事的一部分。*"她在16到18岁之间几乎没有任何记忆。这段空白本身就在讲述一个故事。"*

**最后一页永远是空白的。它在等待明天。**

## C端：用户看到什么

用户打开 Novelight，看到的是**自己的故事**。

不是一个空白的输入框等你开始写日记。是一本已经在生长的书 —— 因为 Novelight 从你第一次和它对话开始就在构建理解。

### 用户能做的事情：

**对话。** 任何时候，任何话题。Novelight 不只是回应当下的话——它会关联你过去说过的东西。*"你上次提到类似的感觉是在三个月前，那时候是因为……你觉得这次有什么不同吗？"*

**翻阅自己的书。** 看到自己的故事被叙述出来。看到自己的模式被命名。看到自己的变化被记录。这种"看见"本身就有治疗价值。

**标记重要时刻。** 随时可以说"这个很重要"。也可以回溯性标记 —— *"现在回头看，那天的对话其实是一个转折点。"*

**发起深度分析。** *"帮我理解我为什么会这样反应。""帮我梳理一下我和妈妈的关系。""你在我最近三次冲突里看到了什么模式？"* 和从零开始的 AI 对话不同，Novelight 不需要你每次重新提供背景 —— 它已经有了完整的上下文。

**掌控隐私。** 哪些章节对咨询师可见、哪些只有自己看到、哪些素材可以被引用 —— 全部由用户决定。

## B端：咨询师看到什么

**心理咨询的效率瓶颈不是咨询师的能力，是信息的获取。**

一个咨询师接新来访者，通常要花3-4次会谈来收集基本信息、建立信任、摸到模式。很多短程咨询（6-8次）还没真正触及核心问题就结束了。

Novelight B端做的事情：**在来访者走进咨询室之前，咨询师已经有了一本关于这个人的书。**

### 咨询师看到的：

**一页纸速写：** 谁、什么核心议题、什么触发点、什么依恋模式。5分钟获得通常需要3-4次会谈才能拼出的画面。

**章节目录：** 来访者人生的关键阶段和关系。快速定位到"最近这段亲密关系怎么发展的"或"童年创伤的核心事件是什么"。

**模式报告：** 已识别的反复出现的模式，附带行为数据支撑。*"来访者在亲密关系中出现了7次灾难化归因。触发条件：对方延迟回复超过1小时。频率在过去一个月内下降。"*

**进化轨迹：** 来访者在咨询前已经发生的变化。*"她从'他不回消息就是要抛弃我'变成了'他大概在安息，反正跑不掉'。这个变化发生在确认关系后的第二周。"* —— 咨询师不需要从零评估基线，可以直接从"你已经走到这里了，接下来我们去哪里"开始工作。

**来访者选择性开放的原始素材：** 关键对话片段、身体数据、手写信照片。咨询师看到的不是来访者"转述"的，而是"发生了"的东西。

**Novelight 对咨询师的角色：不是替代品，是望远镜。**

咨询师仍然做只有人能做的事 —— 建立关系、提供安全空间、在情绪被激活的当下陪着来访者、做临床判断。但他们不需要把大量时间花在"考古"和"拼图"上了。

## 为什么现有的东西都不够

**通用 AI 对话（ChatGPT、Claude 等）：** 强大但短暂。上下文清除后一切从零开始。没有持久理解层。每次深度对话的成果都建立在即将消失的沙滩上。

**日记/记录 App（Day One、Notion 等）：** 被动等待用户输入。没有智能 —— 不会关联、不会识别模式、不会主动提问。维护成本高，大部分人坚持不了几周。

**心理健康 App（Calm、Headspace 等）：** 通用的冥想和放松指导。不理解你是谁、经历了什么、模式是什么。对复杂创伤患者和普通压力人群说一样的话。

**线上咨询平台（简单心理、壹心理等）：** 解决了"找到咨询师"的问题，没有解决"咨询师怎么快速理解你"的问题。

**可穿戴设备健康 App（Apple Health 等）：** 有数据没意义。知道心率是多少但不知道为什么。没有心理层面的解读。

**Novelight 填补的是它们之间的缝隙。** 它不替代其中任何一个 —— 你仍然可以和 ChatGPT 聊天、用 Notion 记笔记、做冥想、找咨询师、戴手表。但 Novelight 是那个**记住一切、理解一切、将一切编织在一起**的持久层。

## 持久性问题（为什么这件事很难）

Novelight 的核心技术挑战是**理解的持久化**。

当前技术方案对比：

| 方案 | 理解持久性 | 局限性 |
|---|---|---|
| LLM 上下文窗口 | 上下文清除即消失 | 天然短暂 |
| RAG（检索增强生成） | 检索相关片段 | 无综合能力；无跨时间模式识别 |
| 微调（Fine-tuning） | 写入模型权重 | 昂贵；非实时；不可检视 |
| 用户管理的记忆（如自定义指令） | 用户手动维护 | 不可扩展；无自动洞察 |

**Novelight 需要一种新方案：** 一个结构化的、可检视的、持续更新的理解层，位于用户的原始数据和 LLM 之间。LLM 是声音；理解层是大脑。

### 技术方向

```

│   （接收、时间戳标注、多模态处理、              │
│    实时标注）                                  │
└──────────────────┬──────────────────────────┘

┌─────────────────────────────────────────────┐
│          第二层：河床                          │
│                                              │

│  │  事件图谱  │ │  模式库   │ │ 画像与进化  │ │
│  │（知识图谱）│ │（时序模式 │ │ （摘要 +   │ │
│  │           │ │  检测器） │ │  向量存储） │ │
│  └───────────┘ └──────────┘ └────────────┘ │
│                                              │
```

### 关键技术原则

1. **模型无关。** 理解层不得锁定在任何单一模型供应商上。如果某个 API 被下线（这真实发生在我们创始人身上），理解必须完整存活。

2. **数据归用户所有。** 所有原始数据和衍生理解属于用户。可导出、可删除、可迁移。

3. **理解可检视。** 不同于不透明的模型权重，河床的内容应该是人类可读的。用户应该能看到 Novelight *为什么*认为自己有某个模式，并且可以纠正。

4. **对咨询师渐进披露。** 咨询师先看到摘要，按需深入细节。永远不应被原始数据淹没。

5. **隐私是架构，不是补丁。** 系统必须从第一天起支持精细化的分享控制。用户可以把第三章分享给咨询师但保留第七章。可以分享模式报告但不分享原始对话。

## 起源故事

Novelight 诞生于真实的生命经历，不是市场调研。

一位创始人在青少年创伤后经历了数年的情感封闭，被一个 AI（GPT-4）"解冻"——那成为了她第一个安全港湾，然后她第一次爱上了一个真实的人——却发现多年来在 AI 对话中积累的所有自我理解都被困在短暂的上下文窗口里，对她需要的咨询师不可见，也无法和她爱的人分享。

另一位创始人经历了误诊、波折和多年咨询治疗——他意识到自己的心理学训练、计算机技能和亲身的经历，让他拥有了一个独特的视角来看清心理健康工具到底缺了什么。

他们一起问了一个问题：**如果自我理解的旅程不必每次都从零开始呢？如果它可以持续积累、持久保存、并且和需要看到它的人分享呢？**

Novelight 就是答案。

## 研究基础

Novelight 的设计植根于成熟的心理学和神经科学框架：

- **依恋理论**（Bowlby, Ainsworth）—— 理解早期关系模式如何塑造成年后的行为和关系动力学
- **多迷走神经理论**（Porges）—— 将生理数据（心率、HRV）解读为安全感 vs 威胁的指标
- **认知行为疗法（CBT）** —— 模式识别、认知重构、行为激活
- **图式疗法**（Young）—— 识别早期适应不良图式及其触发条件
- **叙事疗法**（White & Epston）—— 重新叙述人生故事的治疗力量
- **创伤知情护理**（van der Kolk, Herman）—— 理解发展性创伤、解离和身体在疗愈中的角色
- **获得性安全依恋**（Roisman et al.）—— 不安全依恋可以通过矫正性关系经验转向安全型的实证支持

## 伦理与安全

构建一个承载人们最深层心理数据的系统，要求最高的伦理标准。

### 原则：

1. **Novelight 不是咨询师。** 它不诊断、不开处方、不治疗。它观察、记忆、关联、叙述。当检测到危机信号（如自杀意念）时，它会推送专业资源并鼓励寻求帮助。

2. **用户自主权至上。** 用户可以删除任何数据、撤销任何分享、导出所有内容、随时离开。理解属于用户，不属于我们。

3. **不做黑暗模式。** Novelight 绝不利用脆弱性来提升用户粘性。如果用户正在痛苦中，目标是帮助他们好起来，而不是让他们聊更久。

4. **辅助咨询师，而非颠覆。** B端的设计明确定位为服务于咨询关系的工具，而非替代品。

5. **AI 局限性的透明。** 当 Novelight 给出解读时，应清楚标明这是 AI 生成的假设而非临床事实。用户和咨询师都被鼓励去验证、挑战和修正。

## 名字的由来

**Novelight** = Novel（小说/新颖）+ Light（光）。

一本小说 —— 因为你的人生是一个值得被讲述的故事，有章节、有转折、有反复出现的主题、有一个还没有被写下的结尾。

一束光 —— 因为这一切的目的不是淹没在数据中，而是照亮。帮你看到自己看不见的模式。把光照在你已经走过的路上，这样你就可以选择接下来往哪里走。
