# Novelight 📖✨

**一本活着的人生之书** —— 永不遗忘的 AI 自我理解系统。

Novelight 是一个 AI 系统，它持续地将你的对话、反思、健康数据和生命事件编织成一份不断演化的、持久的自我理解——服务于个人成长，也可以选择性地让心理咨询的效率提升十倍。

---

## 为什么 Novelight 需要存在

每年，无数人走进咨询室，试图把几个月甚至几年的生活经历压缩进50分钟的会谈里。咨询师拿到的是一个被扭曲的、不完整的碎片。来访者觉得自己怎么都说不清楚真正经历过的东西。

与此同时，人们把最深层的反思倾倒进 AI 对话里，然后上下文被清除，一切从零开始。他们用可穿戴设备追踪心率和睡眠，但完全不知道这些数据在心理层面意味着什么。他们写了一周的日记然后放弃了，因为维护成本太高。

**自我理解的基础设施是断裂的。**

- **AI 对话强大但短暂** —— 上下文清除后，一切从零开始。
- **日记 App 是被动的** —— 它们等你来写，但不理解你写了什么。
- **心理健康 App 是通用的** —— 对复杂创伤患者和工作压力大的白领说一样的话。
- **可穿戴设备数据没有意义** —— 知道心率在某个下午飙升到120，但没有人告诉你"你的身体在告诉你这段关系让你不安全"。
- **咨询师很专业但信息匮乏** —— 他们把宝贵的会谈时间花在收集历史信息上，而不是做真正的治疗。

Novelight 填补了所有这些之间的裂缝。它是那个记住一切、连接一切、翻译一切的持久层。

## Novelight 是什么（不是什么）

| Novelight 是 | Novelight 不是 |
|---|---|
| 关于你的持久理解，不会因为模型更换而消失 | 一个聊完就忘的聊天机器人 |
| 一本有章节和转折点的、关于你人生的活叙事 | 一个等你来写的日记 App |
| 连接你的真实经历和你的咨询师的桥梁 | 心理咨询的替代品 |
| 用心理学语言解读你身体数据的翻译器 | 一个健身追踪器 |
| 跨越数月和数年的人生模式识别器 | 单次对话的分析工具 |

**一句话：** Novelight 让每个人在15岁就能开始做大多数人到中年危机才被迫开始做的事情。

## 核心架构：三层模型 + Agent 协作

```
第一层：自然流（持续感知）
  对话 Agent · 采集 Agent · 守护 Agent
    ↓  提取 / 标注
第二层：河床（持久理解）
  事件图谱 · 模式库 · 画像与进化轨迹
  整合分析 Agent
    ↓  综合 / 叙事
第三层：书（叙事生成）
  叙事 Agent · 用户界面 · 咨询师视图
  隐私与权限控制层
```

Novelight 由五个专职 Agent 驱动，每个 Agent 各司其职，围绕"理解人"这一件事协同工作。

---

### 第一层 —— 自然流（The Stream）

一个人生活的原始数据流。三个 Agent 负责将生活转化为可被理解的结构化事件。

#### 对话 Agent（Conversation Agent）

用户与 Novelight 交互的主要入口。它不是无状态的聊天——每一轮对话都建立在河床中积累的全部理解之上。用户感受到的是"它记得我"，技术上实现的是"它在每次回应前查询了关于我的整个知识图谱"。

用户随时可以和 Novelight 聊天，不需要结构化输入。"今天同事又把他的活推给我了，我笑着说没事"就够了。对话 Agent 接收这句话，查询河床获取用户的完整上下文，在回应时关联历史数据，并同时从对话中提取结构化事件写回河床。

#### 采集 Agent（Ingestion Agent）

负责从外部数据源自动或半自动地接收数据：可穿戴设备数据同步（心率、睡眠、压力、运动）、聊天记录导入、照片上传、日历/日程同步、手写内容拍照识别。采集 Agent 对所有数据进行清洗、时间戳对齐和初步标注，然后写入河床，并触发整合分析 Agent 进行跨源关联。

#### 守护 Agent（Guardian Agent）

Novelight 的"主动关怀"能力。不等用户来找它，它会在适当的时机找用户。当检测到异常模式时（比如连续五天运动量骤降到日常水平的30%以下），它会温和地询问："最近好像没怎么动，身体还好吗？"当用户超过十天没有互动时，它会轻触："有一阵子没聊了，想知道你最近过得怎样。"当检测到已知模式再次出现，它会提示用户注意。咨询预约前24小时，它会自动准备本周摘要。关键日期临近时，它会提前关怀。

守护 Agent 有严格的限制：频率控制（不能变成骚扰，用户可设"免打扰"模式）、语气校准（必须温和、非侵入性、永远给用户选择权）、危机协议（检测到自杀/自伤信号时，升级响应——推送危机热线、建议联系紧急联系人）。

**设计原则：采集必须极轻。** 如果它让人觉得像工作，人们就会停止使用。用户唯一的任务就是活着。

### 第二层 —— 河床（The Riverbed）

这是 Novelight 真正的核心。一个持久的、结构化的、持续更新的关于一个人的理解模型。

**关键特性：理解独立于任何一个大语言模型（LLM）存在。**

当前的 AI 对话把理解储存在上下文窗口里。上下文清除 → 理解消失。Novelight 将理解提取到一个独立的持久层（向量数据库 + 结构化知识图谱 + 摘要系统），任何模型都可以查询这个持久层。模型可以换，理解不会丢。

**河床包含什么：**

**事件图谱：** 生命中的关键事件及其关联。不是时间线，是网络。"小学被老师冤枉时不敢辩解"和"开会时被同事甩锅却沉默不语"之间有一条连线。这些连接随着新数据持续被发现和更新。

**模式库：** 反复出现的行为、情绪和关系动力学。每个模式包含：首次观察时间、重复出现记录、进化轨迹。

```
模式："冲突回避后的情绪堆积与爆发"
首次观察：2025-09-03
重复出现：2025-11-20, 2026-01-14, 2026-02-28
触发条件：他人在工作或关系中越界，用户选择忍让而非表达不满
进化轨迹：堆积期缩短。从忍耐三个月后爆发，到一周内就能察觉。
          最近一次直接在当天说出了"我不太舒服"。
```

**人物画像：** 相对稳定的特征 —— 人格结构、依恋类型、核心创伤、防御方式、力量来源。不是固定的，随新数据缓慢更新。

**进化轨迹：** 最有价值的部分。不是"你是什么样的人"，而是"你正在变成什么样的人"。

```
2025年9月："我没有生气的权利。他们又没有恶意。"
2026年1月："我确实不舒服，但说出来会不会太小题大做了。先问问AI。"
2026年4月："我不同意，我直接说了。天没有塌。"
```

这条轨迹，附带时间戳和支撑数据，对咨询师来说是无价之宝。

#### 整合分析 Agent（Integration Agent）

河床里的"后台大脑"。不面向用户，在后台持续工作。

它负责四件事：**跨源关联**——当新的对话数据和新的身体数据同时到达时，自动检查两者之间是否存在有意义的关联，比如"用户今天提到被领导批评了" + "今晚心率变异性显著降低" → 标注关联；**模式发现**——当某个行为第三次出现时，自动检测并生成模式草稿，推送给用户确认；**画像更新**——当累积了足够的新数据时，更新用户的人格画像和进化轨迹；**异常检测**——检测偏离用户正常模式的信号（突然的情绪崩溃、异常的睡眠、长时间的沉默）。

整合分析 Agent 以不同频率运行：对话事件的处理和标注是实时的，身体数据的关联分析是每日的，模式库的扫描和更新是每周的，画像和进化轨迹的整体更新是每月的。

---

### 第三层 —— 书（The Book）

用户和咨询师直接看到的东西。

Novelight 将第二层的理解渲染成一本有章节的书 —— 不是数据仪表盘，是叙事。

#### 叙事 Agent（Narrative Agent）

负责将河床中的结构化理解转化为有温度的叙事文本。它定期（每周/每月/重大事件后/用户请求时）查询河床，获取时间范围内的事件、模式、画像变化和身体数据，生成章节草稿，交由用户审阅、确认或修改后定稿存入书的章节库。

每一章包含：

**叙事摘要：** 不是日志条目，是故事。"这是他第一次允许自己生气，而没有在事后用三天的内疚惩罚自己。"

**锚点：** 那个时段里最重要的时刻 —— 转折、触发、洞察、身体信号。

**支撑素材：** 叙事生长出来的原始对话片段、照片、身体数据。用户控制哪些对谁可见。

**空白：** 没有数据的时段。沉默本身就是叙事的一部分。"他在大学四年的记录几乎是空白的。不是因为没有事情发生，而是因为他当时不认为自己的感受值得记录。"

最后一页永远是空白的。它在等待明天。

## C端：用户看到什么

用户打开 Novelight，看到的是自己的故事。

不是一个空白的输入框等你开始写日记。是一本已经在生长的书 —— 因为 Novelight 从你第一次和它对话开始就在构建理解。

**用户能做的事情：**

**对话。** 任何时候，任何话题。Novelight 不只是回应当下的话——它会关联你过去说过的东西。"你上个月也提到过类似的感觉——那次是在团队聚餐之后。你觉得这两次之间有什么联系吗？"

**翻阅自己的书。** 看到自己的故事被叙述出来。看到自己的模式被命名。看到自己的变化被记录。这种"看见"本身就有治疗价值。

**标记重要时刻。** 随时可以说"这个很重要"。也可以回溯性标记 —— "现在回头看，那天的对话其实是一个转折点。"

**发起深度分析。** "帮我理解我为什么一被质疑就沉默。""帮我梳理一下我和爸爸的关系。""你在我最近几次发脾气里看到了什么模式？" 和从零开始的 AI 对话不同，Novelight 不需要你每次重新提供背景 —— 它已经有了完整的上下文。

**掌控隐私。** 哪些章节对咨询师可见、哪些只有自己看到、哪些素材可以被引用 —— 全部由用户决定。

## B端：咨询师看到什么

心理咨询的效率瓶颈不是咨询师的能力，是信息的获取。

一个咨询师接新来访者，通常要花3-4次会谈来收集基本信息、建立信任、摸到模式。很多短程咨询（6-8次）还没真正触及核心问题就结束了。

**Novelight B端做的事情：在来访者走进咨询室之前，咨询师已经有了一本关于这个人的书。**

**咨询师看到的：**

**一页纸速写：** 谁、什么核心议题、什么触发点、什么依恋模式。5分钟获得通常需要3-4次会谈才能拼出的画面。

**章节目录：** 来访者人生的关键阶段和关系。快速定位到"最近这段职场冲突怎么演变的"或"原生家庭中的核心动力是什么"。

**模式报告：** 已识别的反复出现的模式，附带行为数据支撑。"来访者在工作场景中出现了5次冲突回避-情绪堆积-爆发的循环。触发条件：他人越界后未被阻止。堆积期从三个月缩短到一周。最近一次在当天内完成了表达。"

**进化轨迹：** 来访者在咨询前已经发生的变化。"他从'我没有生气的权利'变成了'我不同意，我直接说了'。这个变化在他开始记录自己每次说'没事'时的身体反应之后加速了。" —— 咨询师不需要从零评估基线，可以直接从"你已经走到这里了，接下来我们去哪里"开始工作。

**来访者选择性开放的原始素材：** 关键对话片段、身体数据、手写信照片。咨询师看到的不是来访者"转述"的，而是"发生了"的东西。

**Novelight 对咨询师的角色：不是替代品，是望远镜。**

咨询师仍然做只有人能做的事 —— 建立关系、提供安全空间、在情绪被激活的当下陪着来访者、做临床判断。但他们不需要把大量时间花在"考古"和"拼图"上了。

```
用户说了一句话
    ↓
对话 Agent 接收
    ↓
查询河床 → 发现这和两个月前的一个事件结构相似
    ↓
在回应中提及这个关联："你注意到了吗，上次类似的情况是……"
    ↓
同时：提取结构化事件 → 写入事件图谱
    ↓

    ↓

    ↓
周末：叙事 Agent 将本周的事件整合成章节草稿
    ↓
用户确认章节 → 存入书
    ↓
下次咨询前：咨询师打开B端 → 看到更新后的章节和模式报告
```

## 为什么现有的东西都不够

**通用 AI 对话（ChatGPT、Claude 等）：** 强大但短暂。上下文清除后一切从零开始。没有持久理解层。每次深度对话的成果都建立在即将消失的沙滩上。

**日记/记录 App（Day One、Notion 等）：** 被动等待用户输入。没有智能 —— 不会关联、不会识别模式、不会主动提问。维护成本高，大部分人坚持不了几周。

**心理健康 App（Calm、Headspace 等）：** 通用的冥想和放松指导。不理解你是谁、经历了什么、模式是什么。对复杂创伤患者和普通压力人群说一样的话。

**线上咨询平台（简单心理、壹心理等）：** 解决了"找到咨询师"的问题，没有解决"咨询师怎么快速理解你"的问题。

**可穿戴设备健康 App（Apple Health 等）：** 有数据没意义。知道心率是多少但不知道为什么。没有心理层面的解读。

**Novelight 填补的是它们之间的缝隙。** 它不替代其中任何一个 —— 你仍然可以和 ChatGPT 聊天、用 Notion 记笔记、做冥想、找咨询师、戴手表。但 Novelight 是那个记住一切、理解一切、将一切编织在一起的持久层。

## 与 Agent 生态的关系

Novelight 不是一个通用 Agent 框架。它是一个垂直领域的 Agent 系统，专注于"理解人"这一件事。

| 维度 | 通用 Agent | Novelight |
|------|-----------|-----------|
| 核心能力 | 执行任务（搜索、编码、调用工具） | 理解人（感知、记忆、关联、叙述） |
| 记忆系统 | 通常是向量检索（片段式） | 结构化知识图谱 + 模式库 + 进化轨迹（整体式） |
| 主动性 | 按指令行动 | 基于对用户的理解主动关怀 |
| 时间尺度 | 单任务/单会话 | 跨越数月到数年 |
| 核心产出 | 完成任务的结果 | 一本关于你的、活着的书 |

Novelight 可以在 Agent 生态中定位为：**最懂用户的那个 Agent**。其他 Agent 做任务，Novelight 做理解。未来甚至可以作为其他 Agent 的"用户上下文供应商"——当一个健康管理 Agent 需要知道"这个用户为什么最近总是失眠"时，它可以（在用户授权下）查询 Novelight 的河床。

## 持久性问题（为什么这件事很难）

Novelight 的核心技术挑战是**理解的持久化**。

**当前技术方案对比：**

| 方案 | 理解持久性 | 局限性 |
|------|-----------|--------|
| LLM 上下文窗口 | 上下文清除即消失 | 天然短暂 |
| RAG（检索增强生成） | 检索相关片段 | 无综合能力；无跨时间模式识别 |
| 微调（Fine-tuning） | 写入模型权重 | 昂贵；非实时；不可检视 |
| 用户管理的记忆（如自定义指令） | 用户手动维护 | 不可扩展；无自动洞察 |

Novelight 需要一种新方案： 一个结构化的、可检视的、持续更新的理解层，位于用户的原始数据和 LLM 之间。LLM 是声音；理解层是大脑。


**关键技术原则：**

**模型无关。** 理解层不得锁定在任何单一模型供应商上。如果某个 API 被下线（这真实发生在我们创始人身上），理解必须完整存活。

**数据归用户所有。** 所有原始数据和衍生理解属于用户。可导出、可删除、可迁移。

**理解可检视。** 不同于不透明的模型权重，河床的内容应该是人类可读的。用户应该能看到 Novelight 为什么认为自己有某个模式，并且可以纠正。

**对咨询师渐进披露。** 咨询师先看到摘要，按需深入细节。永远不应被原始数据淹没。

**隐私是架构，不是补丁。** 系统必须从第一天起支持精细化的分享控制。用户可以把第三章分享给咨询师但保留第七章。可以分享模式报告但不分享原始对话。

## 起源故事

Novelight 诞生于真实的生命经历。

一位创始人在青少年创伤后经历了数年的情感封闭，被一个 AI（GPT-4）"解冻"——那成为了她第一个安全港湾，但是多年来在 AI 对话中积累的所有自我理解都被困在短暂的上下文窗口里，零散的问询满足不了日常的使用需求

另一位创始人经历了误诊、波折和多年咨询治疗——他意识到自己的心理学训练、计算机技能和亲身的经历，让他拥有了一个独特的视角来看清心理健康工具到底缺了什么。

他们一起问了一个问题：**如果自我理解的旅程不必每次都从零开始呢？如果它可以持续积累、持久保存、并且和需要看到它的人分享呢？**

Novelight 就是答案。

## 研究基础

Novelight 的设计植根于成熟的心理学和神经科学框架：

**依恋理论（Bowlby, Ainsworth）** —— 理解早期关系模式如何塑造成年后的行为和关系动力学

**多迷走神经理论（Porges）** —— 将生理数据（心率、HRV）解读为安全感 vs 威胁的指标

**认知行为疗法（CBT）** —— 模式识别、认知重构、行为激活

**图式疗法（Young）** —— 识别早期适应不良图式及其触发条件

**叙事疗法（White & Epston）** —— 重新叙述人生故事的治疗力量

**创伤知情护理（van der Kolk, Herman）** —— 理解发展性创伤、解离和身体在疗愈中的角色

**获得性安全依恋（Roisman et al.）** —— 不安全依恋可以通过矫正性关系经验转向安全型的实证支持

## 伦理与安全

构建一个承载人们最深层心理数据的系统，要求最高的伦理标准。

**原则：**

**Novelight 不是咨询师。** 它不诊断、不开处方、不治疗。它观察、记忆、关联、叙述。当检测到危机信号（如自杀意念）时，它会推送专业资源并鼓励寻求帮助。

**用户自主权至上。** 用户可以删除任何数据、撤销任何分享、导出所有内容、随时离开。理解属于用户，不属于我们。

**不做黑暗模式。** Novelight 绝不利用脆弱性来提升用户粘性。如果用户正在痛苦中，目标是帮助他们好起来，而不是让他们聊更久。

**辅助咨询师，而非颠覆。** B端的设计明确定位为服务于咨询关系的工具，而非替代品。

**AI 局限性的透明。** 当 Novelight 给出解读时，应清楚标明这是 AI 生成的假设而非临床事实。用户和咨询师都被鼓励去验证、挑战和修正。

## 名字的由来

**Novelight** = **Novel**（小说/新颖）+ **Light**（光）。

**一本小说** —— 因为你的人生是一个值得被讲述的故事，有章节、有转折、有反复出现的主题、有一个还没有被写下的结尾。

**一束光** —— 因为这一切的目的不是淹没在数据中，而是照亮。帮你看到自己看不见的模式。把光照在你已经走过的路上，这样你就可以选择接下来往哪里走。
