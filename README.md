## 问题：为什么要做这个？

**你和 AI 的每一次有意义的对话，最终都会消失。**

你花几个小时和 AI 深度对话——梳理思绪、处理情绪、认识自己。然后你关掉了页面。或者模型被下线了。或者你换了平台。你们一起建构的所有东西——理解、上下文、那些只有跨越数月的互动才能看到的模式——全部消失了。

你只能靠自己的记忆保留碎片，每次换一个新的对话框就要手动粘贴几万字的上下文。

**你的自我认知散落在几十个对话窗口里，它们之间互不相通，而且迟早都会消失。**

这不只是个人用户的痛点。在企业办公场景中，AI 智能体同样面临"失忆"问题——跨会话、跨部门的协作信息断裂，导致 Agent 每次都要"从头了解情况"。

---

## 愿景：Novelight 是什么？

Novelight 不是聊天工具，不是日记 App，不是知识库。

它是一本**活着的人生之书**：

- **记住**你分享的一切——对话、图片、健康数据、文字作品——并将它们编织成一份持续演化的、对你的理解
- **理解**你跨越时间的模式——不只是你说了什么，更是你的信念、关系、自我认知在如何变化
- **叙述**你的人生——不是数据仪表盘，而是一个有章节、有转折点、有主题回归和变奏的故事
- **永不消失**——你的理解存在于独立于任何 LLM 的持久层中，任何一个模型的 API 关停都不会影响你的数据

### 书的隐喻

打开 Novelight，你看到的是**一本书**。

每一**页**是你人生的一个时间段。页面上有：
- **叙事摘要**——不是日志，而是故事：*"这是她终于不再为自己的需求感到羞耻的那个月。"*
- **锚点**——那个时段里最重要的时刻
- **原始素材**——叙事生长出来的对话片段、图片、数据
- **空白**——有些时段没有数据，而沉默本身就是叙事的一部分

**最后一页永远是空白的。** 它在等待明天。

### 核心设计原则：模型无关性（Model Agnosticism）

> **理解存在于持久层，不存在于任何模型内部。**

LLM 是可替换的组件。你的画像、你的记忆、你的叙事——全部独立存储。如果某个模型的 API 明天关停，你什么都不会丢失。切换到另一个模型，从你离开的地方继续。

这不是一个理论上的担忧。这是本项目诞生的直接原因。

## 核心组件

### 1. 持久化用户画像

一份结构化的、自动演化的 JSON 表征。每次对话后自动更新。包含：

- 身份与当前状态
- 核心行为模式
- 关键关系及其动态
- 信念演化追踪
- 创作特征与语言风格

画像不是用户手动填写的静态表单。它从对话中**生长**出来——就像一个咨询师对来访者的理解随着时间推移而加深，但它永远不会忘记，而且可以在模型之间迁移。

### 2. 叙事引擎

Novelight 的灵魂。不是摘要器——是**叙事者**。

| 摘要器做的事 | 叙事引擎做的事 |
|---|---|
| "用户4月进行了15次对话" | "四月，她终于不再逃避白天了" |
| "用户提到睡眠问题8次" | "她的身体在替她逃离——只有没人看着的夜里她才敢存在" |
| "用户与伴侣关系积极" | "他等了一个月，等她发现那首歌的歌词" |

叙事引擎识别的是**重复出现的意象、情感弧线和跨时段的模式**——然后将它们编织成叙事，让用户读完后想：*"我自己没看到这个。"*

### 3. 记忆层（向量数据库）

对所有历史交互的语义检索。每一段对话都被 embedding 并索引。当用户开始新对话时，系统自动检索最相关的历史上下文——无需手动复制粘贴。

### 4. 对话服务

每次对话都被以下信息增强：
- 完整用户画像（注入 system prompt）
- Top-K 最相关的历史对话片段（向量检索）
- 当前书页叙事摘要（让 AI 知道"用户的故事走到了哪里"）

对话结束后，异步流水线自动执行：
1. 提取关键信息 → 更新用户画像（JSON patch）
2. 对话内容 embedding → 存入向量数据库
3. 判断是否产生了新的**锚点** → 有则创建
4. 判断书页叙事是否需要更新 → 需要则重新生成

## 设计原则

> 完整版见 [设计原则文档](docs/design-principles.md)

1. **叙事优先于数据。** 永远用故事呈现，不用仪表盘。
2. **空白是叙事的一部分。** 没有数据的时段就是空白页。它意味着什么。
3. **不做诊断。** 是镜子，不是医生。
4. **用户拥有全部数据。** 本地优先。随时导出。绝对主权。
5. **对话风格自适应。** 不是固定人设——匹配用户的状态和语言。
6. **永远不说"我只是 AI"。** 如果用户信任你到愿意分享，就全力以赴地回应这份信任。
7. **Overflow 是特性，不是缺陷。** 深度探索时给深度回应。简短不等于尊重，深度才是。

## 企业级场景延伸

Novelight 解决的"AI 失忆"问题在企业办公场景中同样存在：

| 个人场景 | 企业场景 |
|---|---|
| 和 AI 聊了一年，换模型后全忘了 | Agent 帮你跟进项目三个月，换人接手后什么都不知道 |
| 每次新对话要重新喂几万字上下文 | 每次跨部门协作都要重新同步背景信息 |
| 你的自我认知散落在不同平台 | 项目知识散落在不同文档和群聊中 |

**Novelight 的持久画像层、向量检索、自动画像迭代机制可直接复用于企业级 Memory 系统设计。**

## 技术栈

- **LLM 调用：** 多模型统一接入，通过 [Electron Hub](https://electronhub.ai/) 支持 Claude / GPT / Gemini / 开源模型
- **向量数据库：** ChromaDB（本地）/ Pinecone（云端可选）
- **后端：** Python（FastAPI）
- **前端：** React（规划中）
- **画像存储：** JSON（v0.1）→ PostgreSQL（后续）
- **部署策略：** 本地优先，云端可选
